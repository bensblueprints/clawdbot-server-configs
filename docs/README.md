## Contributing Documentation

When adding documentation:

1. Use Markdown format (.md files)
2. Include code examples where applicable
3. Add a table of contents for long documents
4. Link related documents together
5. Keep language clear and concise

## Documentation Standards

- **Headings:** Use proper hierarchy (# > ## > ###)
- **Code blocks:** Specify language for syntax highlighting
- **Commands:** Show both command and expected output
- **Examples:** Provide real-world scenarios
- **Updates:** Include "Last Updated" date
